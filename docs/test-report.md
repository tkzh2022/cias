# 测试与验证报告

## 测试范围
- 后端单元测试：`ChatService` 转人工与链路行为。
- 后端接口测试：`/api/chat` 响应结构与状态码。
- 前端构建验证：TypeScript 编译 + Vite 打包。
- 压测脚本：k6 并发/时延阈值验证脚本。

## 结果记录（本地）
- `cd cs-server && mvn test`：用于验证后端核心逻辑和接口层。
- `cd cs-web && npm run build`：用于验证前端可编译和可打包。
- `k6 run tests/k6/chat-load-test.js`：用于验证时延阈值和失败率阈值。

## 缺陷闭环
- 发现编译、依赖、接口错误后，立即修复并重新回归。
- 目标状态：无阻断级（P0/P1）缺陷；核心路径可重复通过。

## 已知限制与后续优化
- 当前检索器与模型调用使用演示级实现，生产需接入真实向量库与模型网关。
- 可新增指标：缓存命中率、路由命中率、token 成本统计。
