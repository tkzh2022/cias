# 智能客服市场实现方式调研

## 主流实现路径
- 规则引擎客服：实现快、成本低，但泛化能力弱、维护成本高。
- 传统 NLP（意图识别+槽位填充）：在固定场景稳定，复杂多轮对话表现一般。
- RAG 客服：通过检索外部知识作为事实依据，兼顾准确率与可更新性。
- Agent 客服：可调用工单、订单、CRM 等工具，适合复杂任务闭环。
- 多模型路由：按问题复杂度和成本/延迟目标动态选模型，优化性价比。

## 高并发低延迟共性实践
- 缓存优先：FAQ缓存、语义缓存、会话缓存，优先走毫秒级命中路径。
- 异步化：日志、反馈学习、知识库增量处理放入消息队列。
- 推理优化：模型路由、请求超时、重试、熔断、降级与转人工兜底。
- 可观测性：QPS、P95、错误率、token 成本、模型命中率全链路监控。

## 本项目选型结论
- 核心方案：`RAG + 多模型路由 + Redis缓存 + Kafka异步 + 熔断降级`。
- 交付路径：先构建可运行闭环，再通过压测定位瓶颈持续优化。
